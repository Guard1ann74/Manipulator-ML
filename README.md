# Разработка системы управления роботизированным манаипулятором на естественном языке  

Проект по реализации интеллектуального интерфейса управления роботом-манипулятором. Основная идея заключается в использовании компактной модели для распознавания текстовых команд и их интеграции в программный код через SDK.

## Описание проекта
В ходе разработки пришел к решению, которое изначально намечалось как приоритетное: взять готовую малую языковую модель и внедрить её непосредственно в код.

Особое внимание было уделено работе с данными. После многократного перебора обучающей выборки и калибровки датасета удалось добиться стабильной работы системы и практически полностью устранить ложные срабатывания. Один из главных инсайтов проекта: насколько бы шикарным и неповторимым ни был метод обучения, если на входе «мусорный» датасет, положительного результата ждать не стоит.

## Архитектура модели
Система представляет собой гибридную архитектуру, состоящую из двух этапов:

1.  **Нейросеть (Encoder)**: **RuBERT-tiny2**.
    *   Модель архитектуры **BERT** (Bidirectional Encoder Representations from Transformers), использующая только энкодерную часть.
    *   На выходе формирует компактный **312-мерный вектор**, описывающий семантику команды.
2.  **Классификатор**: **SVC** (Support Vector Classification).
    *   Алгоритм ищет в 312-мерном пространстве оптимальные границы гиперплоскости, которые максимально четко разделяют заданные классы команд.

## Метрики качества
Для оценки работы манипулятора использовался комплексный подход к метрикам:

*   **Accuracy**: Базовая метрика, показывающая процент правильно распознанных команд. Стоит учитывать, что в некоторых сценариях она может быть обманчивой.
*   **Матрица ошибок**: Таблица, где сопоставляются реальные и предсказанные классы. Позволяет увидеть четкое количество верно определенных примеров или их дробную долю.
*   **Precision и Recall**:
    *   **Precision** показывает, насколько можно доверять модели, когда она утверждает, что команда распознана.
    *   **Recall** показывает, как много реальных команд модель способна «уловить» и не пропустить.
*   **F1-Score**: Среднее гармоническое между Precision и Recall. Выбор гармонического среднего вместо арифметического позволяет оценивать модель строже: если при Recall = 1.0 модель имеет Precision = 0.1, F1-score составит 0.18 (в то время как арифметическое среднее было бы 0.55).
*   **Кросс-валидация**: Метод честной оценки качества. Датасет (например, 500 фраз) делится на 5 фолдов по 100 фраз. Модель проходит 5 итераций обучения и тестирования так, чтобы каждый фолд поочередно выступил в роли тестового. Итоговая точность — среднее арифметическое по всем итерациям.
